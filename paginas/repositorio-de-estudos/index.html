  1. Comandos Básicos

    • Console.WriteLine( ) e Console.Write()

      • Quando eu uso o "Console.WriteLine( )" cada frase inserida entre colchetes começará em uma nova linha, mas se eu usar "Console. Writ(e)" se eu usar novamente a nova frase ainda estará na mesma linha.

      Formas de imprimir na CLI com o Console.WriteLine
      • ""

        • Console.WriteLine( "Para escrever na proxima linha \nUtilize o \\n;");

        Console.WriteLine( "Para dar um espaço de ta(bquatro espaços) \tUtilize o \\t;");

        Console.WriteLine( "Para exibir a barra contraria ou então a aspa utilize uma barra contraria antes.\nPara exibir dessa forma \"\" tem a sigle quotation\' e barra contraria \\ e alguns outros; ");

        Console.WriteLine( "Para usar caracteres Unicode utiliza o \\u;");

        // Kon'nichiwa World

        Console.WriteLine( "Por exemplo: \u3053\u3093\u306B\u3061\u306F World! ");

      • @"" - Textual
        • Console.WriteLine( @" Para exibir o texto da forma que é editado ou pré formatado.  

Por exemplo se você colocar um espaço vai exibir um espaço.
Se você escrever na linha de baixo vai exibir na linha abaixo.
Mas para você utilizar alguns caracteres especiais é necessário utilizar a aspa antes deles "";");
    • Comentarios // e /* */:

      • Quando uso para comentar o código, duas barras na frente do texto são para comentário de uma linha. E quando eu uso um texto dentro de uma barra e uma estrela( asterisco) é para comentários de várias linhas ou em um contexto simples "//" é para comentários curtos e "/* */" para comentários mais longos.

    • Variáveis Numéricas int/ long/ float/ double/ decimal:

      • Para declarar essas variáveis ​​existem algumas observações para tornar o código um "código limpo".

      int E double NÃO PRECISA COLOCAR UMA LETRA APÓS O NÚMERO

      int numInt = 15;

      double numDouble = 12.7; // But here if you would like to be more judicious use 12.7D;

      MAS long, float e decimal PRECISA COLOCAR UMA LETRA APÓS O NÚMERO

      long numLong = 134L;

      float numFloat = 1.3457F;

      decimal numDecimal = 1.78m;

    • Concatenando strings

      • To concatenate strings in C# use the plus simbol `+` :

      string primeiroNome = "Mateus";

      string saudacao = "Hello";

      string mensagem = saudacao + " " + primeiroNome + "!";

      Console.WriteLine (>mensagem);

      //ou

      Console.WriteLine( saudacao + " " + primeiroNome + "!");

    • Interpolando as Variáveis

      • Para interpolar as variáveis, use o cifrão antes das aspas e insira as variáveis ​​entre colchetes:

      string mensagem = $"{saudacao} {primeiroNome}!";

      string mensagem = $"Hello {primeiroNome}";

      Console.WriteLine( mensagem);

      Console.WriteLine( $"{saudacao} {primeiroNome}!");

      Combinando texto "@" e interpolando as variaveis ao mesmo tempo:
      $@""

      string nomeProjeto = "Primeiro-Projeto";

      string diretorioProjeto = $@"C:\User\Mateus\Docs\Projetos\{nomeProjeto}";

    • Operador Mais "+"

      • Para variáveis numericas( int, double) funciona como soma matematica:

      int primNum = 15;

      int secNum = 7;

      Console.WriteLine( $"Resultado: {primNum + secNum}");

      Para variáveis literais e e para variáveis literais mais variáveis numerica funciona como operador de concatenação:

      string primNome = "Mateus";

      int widgetsSold = 11;


      //Aqui o C# vai interpretar toda a impressão como uma string, pois vai converter de forma implicita no código.

      Console.WriteLine( primNome + " vendeu " + widgetsSold + " widgets.");


      //Nesse caso o compilador vai converter o numero em uma string aqui também, pois o compilador não consegue interpretar como um Humano.

      Console.WriteLine( primNome + " vendeu " + widgetsSold + 7 + " widgets.");


      //Mas aqui com o parenteses o compilador vai interpretar que tem que efetuar a soma antes de converter o numero em string.

      Console.WriteLine( primNome + " vendeu " +( widgetsSold + 7) + " widgets.");

      * Nota: O C# os parenteses como em funções mátematicas, primeiro resolve a função dentro do parenteses para depois resolver o resto.
